Introduction

The SafeFileGuard API provides programmatic access to our malware detection capabilities. You can use our API to scan files and URLs for potential threats, and integrate these features into your own applications and workflows.

All API endpoints return data in JSON format and use HTTPS for security.

Authentication

To use the SafeFileGuard API, you need an API key. Each request must include your API key in the HTTP header:

Authorization: Bearer YOUR_API_KEY

To obtain an API key, please contact our support team.

File Scan API

Use this endpoint to scan files for malicious content.

POST /scan/file
Request Parameters

Send the file as a multipart/form-data upload with the key name 'file'.

Example Request
curl -X POST https://api.safe-file-guard.com/scan/file \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -F "file=@/path/to/file.exe"
Example Response
{
    "is_malicious": true,
    "message": "Malware detected: Trojan.GenericKD.45776123",
    "details": {
        "file_name": "file.exe",
        "file_size": 245760,
        "file_type": "PE32 executable",
        "sha256": "8a9f7d3c5b6a1e2d4f5g6h7j8k9l0m1n2o3p4q5r6s7t8u9v0w",
        "threat_type": "Trojan",
        "threat_name": "Trojan.GenericKD.45776123",
        "confidence": "high",
        "signatures_matched": ["suspicious_imports", "known_malware_pattern"]
    },
    "scan_id": "a1b2c3d4-e5f6-7g8h-9i0j-k1l2m3n4o5p6"
}

URL Scan API

Use this endpoint to scan URLs for malicious content.

POST /scan/url
Request Parameters
Parameter Type Description Required
url string The URL to scan Yes
Example Request
curl -X POST https://api.safe-file-guard.com/scan/url \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{"url": "https://example.com/suspicious-page"}'
Example Response
{
    "is_malicious": true,
    "message": "Malicious URL detected: Phishing site",
    "details": {
        "url": "https://example.com/suspicious-page",
        "threat_type": "Phishing",
        "confidence": "high",
        "categories": ["phishing", "malicious_content"],
        "blacklisted": true,
        "related_threats": ["credential_theft"]
    },
    "scan_id": "q7r8s9t0-u1v2-w3x4-y5z6-a7b8c9d0e1f2"
}

Response Codes

The SafeFileGuard API uses standard HTTP response codes to indicate the success or failure of a request.

Code Description
200 Success - The request was processed successfully
400 Bad Request - The request was invalid or improperly formatted
401 Unauthorized - Authentication failed or API key is invalid
403 Forbidden - The API key does not have permission to access the requested resource
404 Not Found - The requested resource was not found
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error - Something went wrong on our end

Rate Limits

The SafeFileGuard API enforces rate limits to ensure fair usage and system stability. Rate limits vary by subscription tier:

Plan Rate Limit
Basic 100 requests per day
Professional 1,000 requests per day
Enterprise Custom limits based on requirements

When you exceed your rate limit, the API will return a 429 response code. The response headers will include information about when you can resume making requests.
