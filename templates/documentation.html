Getting Started

Welcome to the SafeFileGuard documentation. This guide will help you understand how to use our advanced malware detection system effectively. SafeFileGuard uses AI-powered analysis to identify potential threats in files and URLs.

System Requirements

SafeFileGuard is a web-based application that works in any modern browser. For optimal performance, we recommend:

  • Chrome, Firefox, Safari, or Edge (latest versions)
  • JavaScript enabled
  • Stable internet connection

Basic Concepts

SafeFileGuard provides two main scanning capabilities:

  1. File Scanning - Upload and analyze files for malicious content
  2. URL Scanning - Check web addresses for potential threats
Note

All scans are processed securely using our local machine learning models and malware detection algorithms.

Scanning Files

SafeFileGuard allows you to scan various types of files for malicious content. Our system analyzes file structures, code patterns, and behaviors to identify potential threats.

Supported File Types

Our system supports scanning of the following file types:

  • Executables (.exe, .dll, .bat)
  • Documents (.pdf, .doc, .docx, .xls, .xlsx, .ppt, .pptx)
  • Archives (.zip, .rar, .7z)
  • Scripts (.js, .vbs, .ps1, .py)
  • And many more

How to Scan Files

To scan files for malware:

  1. Navigate to the Scanner section on the homepage
  2. Select the "File Scan" tab
  3. Drag and drop files into the upload area or click to browse
  4. Click "Scan Files for Threats"
  5. Wait for the analysis to complete
  6. Review the scan results
Warning

The maximum file size for scanning is 50MB. Larger files will need to be analyzed using our API with appropriate credentials.

Scanning URLs

SafeFileGuard can analyze web addresses to identify potentially malicious websites, phishing attempts, and other web-based threats.

How to Scan URLs

To check if a URL is safe:

  1. Navigate to the Scanner section on the homepage
  2. Select the "URL Scan" tab
  3. Enter the complete URL including http:// or https://
  4. Click "Scan URL"
  5. Wait for the analysis to complete
  6. Review the scan results

Understanding URL Scan Results

URL scan results will indicate whether the URL is:

  • Safe - No threats detected
  • Malicious - URL is likely malicious or contains harmful content
  • Suspicious - Some suspicious elements were detected but not confirmed as malicious

API Usage

SafeFileGuard provides a RESTful API for programmatic access to our scanning capabilities. This allows integration with your own applications and workflows.

Authentication

All API requests require an API key for authentication. To obtain an API key, please contact our support team.

Basic API Request

Here's an example of a basic API request to scan a URL: 

curl -X POST https://api.safefileguard.com/scan/url \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com"}'

For complete API documentation, please visit our API Reference page.
